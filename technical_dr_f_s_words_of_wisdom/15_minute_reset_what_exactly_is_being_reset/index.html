most ecu systems have a learn feature..in its simplist form this is when the lambna sensor in the exhaust measures the oxygen levels and then the ecu applies an adapted fuel value to certain areas of the fuel map to optimise emissions.. if there is, for example an exhaust leak or a dirty filter which the ecu has adapted for by weakening or richening the map . followed by a reset , that adapted map is then eliminated , then you get a missmatch between actual conditions and the exsisting map ...this may be the reason that the re set cleans things up as it will put the map into an unadapted state.
i am no expert in this arena but that is my laymans understanding... i am keen to be put right or learn more ......peace

No it will not reset the akra map to standard. Think of the map like a BIOS in a PC. It tells the hardware how to work. To change these values you have to physically upload a new map to the ECU as you would do a BIOS update.

All it does is reset the adapted figures on the base map (stock, akra, etc) that the ECU has put on due to calculations received from lamba sensor etc.

This is why it is good reson to do it if you make any changes to anything affecting the perfomance of the bike, as it takes it back to the base map.

DribbleDuke

2011-05-17 22:33:00 UTC
Think of it as building a seperate trim map of minor adjustments that it makes on top of the base map that is installed, when you do the reset it just sets all of these trim values to 0 again, but the base map is always the base map and doesn't change without flashing a new base map to the ecu.

If you disable the o2 sensors either with TuneECU or adding o2 eliminators, it for the most part stops any unwanted adjustments to the trim map occurring, this is why you'll see a number of references to o2 elimination on this site as being a potential fix for certain issues.
