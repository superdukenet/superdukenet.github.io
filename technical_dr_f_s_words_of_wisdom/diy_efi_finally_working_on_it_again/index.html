DIY EFI finally working on it again...

joey120373

2010-05-05 18:54:00 UTC
The weather has finally cleared up here in Colorado, and work has eased up a bit, so I thought I would get back to tuning on the Duke...

Last fall I spliced into the stock harness and was able to get it running using a modified MS2/V3.0 board, It was biter sweet though,
sweet cause even with almost no tuning, it revved way faster than on the stock ECU, A quick twist of the throttle and it was bouncing off the rev limmiter before i caould react fast enough to let go of the throttle.
Bitter cause the case for the MS efi was to big to fit under the seat , and the code I wanted to use was not able to control the ignition properly.

Well, now I have built up a smaller board, and the code was fixed ( and upgraded in other areas... ) so I can give it another go.

Since that time I have installed a set of FMF cans, to complement the moto air box that was already on it.

I have also been finishing up the last touches on another PCB, one that will allow PNP "plug and play" replacement of the stock ecu. One of my goals in building this was to have it all cost roughly the same or less that the commonly available tuning scemes ( power comander or tune boy etc. )
So far it looks prommising, I will be spending most of tomorow getting it and a wide band O2 hooked up to do some real tuning. Once I can confirm that everything works ( and hopefully runs bitchen .

The MS2 http://www.msefi.com/viewforum.php?f=64 ... 3046ebafa5
( and microsquirt http://www.microsquirt.com/index.php ) codes offer a wealth of tuning schemes and some very complex fueling and timing algorithms that I am hoping will allow the finicky Vtwin to perfom like a champ, and i was very encouraged by the initial trial , but there is a huge differance between playing around with it in a shop and actually riding it.

The PNP board also has a couple of additional circuits added that should help tuning, one being a MAP mixer that combines the signals from both MAP sensors to give the MS2 a better indication of actual load. The companion project to this is a set of PNP wide band O2 sensors, these will be very small " adapter cables " that contain a Bosch LSU sensor controller that will plug into the stock harness. Quite similar to the Innovate LC-1 only much smaller.

I will try to take some pics to show all the various bits, hopefully i can post those tomorrow as I get the "proto-proto" board installed. It looks pretty "geto" at the moment, mainly cause this is a tepmorary set up, when its all done I plan to have a PNP ECU, dual wide bands, and a touch screen display hooked up. And at this point I have basically no money into it, the small board was a proto i made up years ago fo another project. Once I order the PNP boards though the cost for everything will be getting up there, so an error on the board would suck big time.
The PNP board will cost quite a bit, so I eant to be sure everything will be tits before I order it.

One other thing I have been pondering is somehow incorpeating a MAF (mass air flow) sensor into the air cleaner, this would be ideal as it would give a very accurate indication of engine load at all RPM's and throttle posisions. And the MS2/Microsquirt will use either. as it is, even with crank timing based MAP windowing, and the MAP mixer circuit, engine vaccume is still quite choppy, and dissapears completelly with relatively small throttle angles. This requires a blended scheem if Alpha-n ( TPS vs. RPM ) and Speed Density SD ( MAP vs. RPM ). It can be tuned to run quite well, ( and I am quite certain this is how the stock ECU does it ) but it is tricky, and still not as good as actually measuring the mass of air into the engine.

But first things first, gotta get it up and running and do some serious testing/tuning.

Wish me luck, the way my work schedule is even minor problems or set backs cause weeks of delays .
